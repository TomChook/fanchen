# 凡尘 V3 首屏与核心交互重构设计规格 (Draft v2)

## 1. 设计目标与核心理念

针对当前“太杂乱、操作繁琐”的痛点，彻底抛弃“左常驻状态+右多页签工作区+多浮窗”的古典网页后台式布局。
全新设计理念：**游戏化 HUD 体验、上下文智能联动布局、免维护的关联信息流**。

## 2. 界面信息架构重构

### 2.1 游戏化布局结构 (HUD + Contextual Areas)
- **顶部状态栏 (Top HUD)**：展示核心游戏指标（时间/节气、生命、关键资源如灵石/文钱）。极简化，不放长文字。
- **主操作区 (Main Action Area)**：占据屏幕主要区域。进入不同玩法（修行、产业、市场）时，主操作区发生结构化切换。
- **智能副屏 / 关联面板 (Contextual Sub-panel)**：**【关键改动】绝不使用互斥遮罩完全阻断玩家视线**。考虑到信息关联性，当玩家需要比对信息时（如：在“拍卖行”买东西必须看“背包”和“资金”，在“门派”做事需要看“属性”），系统自动在主操作区合适位置分屏展示关联信息，组合呈现。无需玩家手动开启和拖拽窗口排版。
- **快捷动作栏 / 底部 Dock**：放置高频系统级入口，或核心挂机动作（修练、工作）的快速起停开关。

### 2.2 核心痛点“繁琐”的减负方案
1. **消灭自由窗口，引入导轨钉图 (Widget Pin)**：不再提供自由拖拽的窗口机制。对于玩家需要强行“固定出来”的高频信息（如盯盘某个物品、关注的流言），采用 MMO 式的“屏幕边缘导轨挂载”体系，玩家可将其 Pin 在屏幕左右两侧的固定槽位，不遮挡主操作区。
2. **场景上下文联动**：从“玩家找面板”变成“面板随场景就位”。打开特定场景，所需的关联面板自动铺开。
3. **沉浸化通知**：长篇幅流言和突发事件通过屏幕边缘半透明滚动层或漂浮框展示，尽量不硬切画面，不打断当前联系操作。

## 3. 视觉与交互要求 (类游戏化沉浸体验)
- **材质与氛围**：去除浓重的“后台网页系统”边框和表单感。采用符合修仙题材的设计语言（如暗色、微透蒙版、符合修真气质的留白或高亮体系），增强游戏沉浸感。
- **动效反馈**：数据变化、物品获取时，使用更游戏化的飘字、数字跳动、简单动效等手段反馈，而不是僵硬刷新。

## 4. 落地步骤建议
1. 重写 `App.vue` 级的主要 Layout，确立 “HUD + 动态分屏/主副面板” 结构的骨架。
2. 废弃原有的 `GameWindows.vue` (自由浮窗系统)，改为数据驱动的上下文联动呈现机制。
3. 重构现有业务面板（MarketPanel, InventoryPanel 等），剔除原先带有关闭/拖拽功能的冗余外壳，变为纯净的、随时可被主框架排列的子视图。
