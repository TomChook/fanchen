# 道具视觉与按钮态规范

## 1. 目标

- 让不同品阶的物品在同一面板里一眼可分。
- 让“可点 / 不可点 / 条件不足”的按钮呈现统一反馈。
- 不靠过度发光和廉价特效，维持当前宣纸、水墨、器物陈列的整体气质。

## 2. 品阶视觉语言

| 品阶 | 卡片边界 | 背景方向 | 使用范围 |
|---|---|---|---|
| 凡品 | 低对比灰边 | 浅宣纸色 | 常规材料、低阶装备 |
| 灵品 | 草木青边 | 淡青白渐变 | 基础秘籍、基础产业物资 |
| 玄品 | 水蓝冷边 | 淡水色渐变 | 中段装备、主流功法、进阶材料 |
| 地品 | 金棕暖边 | 温黄纸面 | 宗门级道具、悟道产物、高端书册 |
| 天品 | 朱砂晕染边 | 轻霞色叠层 | 顶级心法、终盘宝物 |

落地规则：

- 不改整体排版，只改卡片边框、底色和局部晕染。
- 稀有度越高，边框越明确，但文本可读性优先，不允许强噪声底图。
- 已装备、已学习、已研读仍用按钮和文案表达状态，不拿品阶色替代状态色。

## 3. 按钮禁用态

### 3.1 统一规则

- 所有不可点击按钮使用同一套禁用态：虚线边框、浅条纹底、文字变灰、鼠标 `not-allowed`。
- 禁用态不再抬升，不再变色，不制造“看着像能点”的误导。
- 若存在明确不足条件，优先通过 `title` 或提示文案给出原因。

### 3.2 典型场景

| 场景 | 按钮文案 | 禁用原因 |
|---|---|---|
| 已学过秘籍 | 已学会 / 已研读 | 重复学习无意义 |
| 当前心法 | 当前心法 | 已经启用 |
| 誊写秘籍但材料不够 | 誊写秘籍 | 缺册页、缺灵墨或真气不足 |
| 材料、耗材原料 | 不露“使用”按钮 | 只展示用途说明，避免误导 |
| 后续扩展中的入口 | 整体隐藏，不露按钮 | 禁止暴露半截功能 |

## 4. 面板应用范围

- 行囊卡片：所有物品卡片使用品阶底色；不能直接使用的材料改为“用途说明 + 出售/收纳”语义。
- 行囊结构：单入口内拆为“行囊 / 修习”双子页，先看到实际背包，再进入功法和学识区。
- 已学功法：卡片同样使用品阶底色，圆满后通过文案和誊抄按钮体现“已解锁”。
- 已学学识：沿用品阶底色，但不放熟练度信息，避免和功法混淆。
- 已装备兵器、护甲、当前心法：保持同样的品阶视觉规则，减少信息割裂。

## 5. 不做的事

- 不给每个品阶单独做浮夸动画。
- 不把按钮禁用态做成完全透明，避免用户误以为功能消失。
- 不用纯黑或重阴影去破坏当前轻宣纸风格。
